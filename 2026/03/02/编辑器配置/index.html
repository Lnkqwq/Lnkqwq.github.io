JSON

JSON语法

JSON（JavaScript Object Notation）是一种轻量级的数据交换格式。它本质上就是一套写配置、传数据的规则——人能看懂，机器也能解析。

核心就两样东西：

· 键值对（对象）
· 列表（数组）

无限套娃，就能构成任何复杂的数据结构。

二、JSON 的两种基础结构

  1. 对象（Object）——键值对的集合

· 用 {} 包起来
· 键必须用双引号
· 键和值之间用冒号 :
· 每对之间用逗号 , 隔开
· 最后一个后面不能有逗号

  1. 数组（Array）——值的有序列表

· 用 [] 包起来
· 值之间用逗号 ,
· 最后一个后面不能有逗号

YAML

YAML语法

一、YAML 是什么

YAML（YAML Ain’t Markup Language）是一种人类友好的数据序列化格式。它的设计目标就是“让人能一眼看懂”。

和 JSON 比：

· JSON：机器友好，严谨，括号多
· YAML：人友好，简洁，靠缩进

核心特点：

· 用缩进表示层级，不用 {} 和 []
· 支持注释
· 字符串可以不写引号
· 配置文件的“首选格式”
