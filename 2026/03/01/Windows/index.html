二、环境变量是什么

  1. 定义

环境变量是操作系统里的一组键值对，告诉系统去哪里找可执行文件、临时文件夹位置等信息。

  1. 最重要的那个：PATH

PATH 是一个列表，里面放着一堆文件夹路径。当你在命令行输入一个命令（比如 g++），系统会按顺序去这些文件夹里找有没有叫 g++.exe 的文件。

Windows 里的斜杠

一、问题来源

  1. 现象

你在不同的地方看到这些写法：

到底哪个是对的？为什么同一个路径要写成不同样子？

  1. 历史原因

· Windows：从 DOS 时代继承，用反斜杠 \ 分隔文件夹
· Unix/Linux：用正斜杠 / 分隔
· Web：URL 里用正斜杠
https://example.com/

你现在 straddle 在三个世界之间，所以乱。

二、各场景应该用什么

场景1：文件资源管理器地址栏

用反斜杠 \

这是 Windows 亲儿子，只能用 \。

场景2：命令行（cmd/PowerShell）

两种都行，但建议用反斜杠

cmd 和 PowerShell 都兼容正斜杠，但传统习惯用反斜杠。

场景3：C/C++ 代码里的字符串

必须用双反斜杠 \ 或正斜杠 /

为什么：C++ 里反斜杠 \ 是转义符的开始，比如 \n 是换行。所以要表示真正的反斜杠，得写两个 \。

场景4：CMakeLists.txt 和 JSON 文件

用正斜杠 /

这些文件里反斜杠容易出问题，用正斜杠最安全。

场景5：相对路径

用正斜杠 /

相对路径里的 … 是语法，分隔符用 / 更清晰，各平台都认。

场景6：Windows API

必须用双反斜杠 \

Windows 底层 API 要求反斜杠，但在 C++ 里要写成双的。

注册表

一、注册表是什么

  1. 一句话定义

Windows 的配置数据库——所有设置、选项、配置都藏在这里。

  1. 类比

· 如果把电脑比作一个人，注册表就是他的大脑+记忆+习惯
· 系统启动时读它，装软件时写它，运行时随时查它
· 改错了，人就傻了

  1. 物理位置（知道就行，别去翻）

二、为什么需要知道注册表

  1. 软件卸载不干净

· 现象：明明卸载了，重装时报“已安装”
· 原因：软件没了，但注册表里还留着它的信息
· 解决：去注册表里删对应键

  1. 开机启动项太多

· 现象：电脑开机越来越慢
· 原因：很多软件把自己写进开机启动
· 解决：去注册表里删掉不想自启的

  1. 右键菜单太乱

· 现象：装了一堆软件，右键菜单比命还长
· 原因：每个软件都往注册表里加自己的右键项
· 解决：去注册表里删掉不用的

  1. 文件关联错了

· 现象：双击 .txt 文件，它用记事本打开，但你想用 VSCode
· 原因：注册表里记录了默认打开方式
· 解决：改注册表或右键“打开方式”

  1. 某些设置界面找不到

· 现象：想改的东西在设置面板里没有
· 原因：微软藏起来了，但注册表里有开关
· 解决：去注册表里改（需要知道具体键值）

三、注册表长什么样

  1. 打开方式

（Win + R 输入 regedit 回车）

  1. 结构（树形，和文件管理器一样）
  1. 五个根键的作用

根键 缩写 存什么
HKEY_CLASSES_ROOT HKCR 文件类型、右键菜单、COM组件
HKEY_CURRENT_USER HKCU 当前用户的设置（主题、壁纸、软件配置）
HKEY_LOCAL_MACHINE HKLM 本机所有用户的设置（硬件、系统、全局软件）
HKEY_USERS HKU 所有用户的配置（一般不用管）
HKEY_CURRENT_CONFIG HKCC 当前硬件配置（一般不用管）

  1. 键和值

· 键（Key）：左边那些文件夹，像 HKEY_CURRENT_USER\Software\Microsoft
· 值（Value）：右边那些，存具体数据
· 值类型：字符串、数字、二进制

四、日常会碰到的几个位置

  1. 开机启动项

右边列出来的，就是开机会自动启动的程序。

  1. 右键菜单（文件）

每个子键对应一个右键菜单项。

  1. 右键菜单（文件夹）
  1. 软件卸载信息

控制面板的“卸载程序”列表，就是从这里读的。

  1. 环境变量（没错，PATH 也在这）

五、操作注册表（必读：先备份！）

  1. 导出备份（改之前一定要做）

· 选中要改的键 → 文件 → 导出
· 存成 .reg 文件
· 万一改坏了，双击这个文件就能恢复

  1. 查找

按 Ctrl+F，输入关键词。但自带的搜索很慢，推荐用 RegScanner（第三方工具）。

  1. 修改

· 右键 → 新建 → 字符串值 / DWORD值 / 等
· 双击改数值

  1. 删除

右键 → 删除（不可恢复，除非有备份）

Windows 系统配置与安全策略”

问题现象

在 PowerShell 里运行 hexo 命令时，出现以下报错：

无法加载文件 …hexo.ps1，因为在此系统上禁止运行脚本。

问题原因

这是 Windows 的 执行策略（Execution Policy） 在起作用。它就像系统的一个安全开关，为了防止恶意脚本运行，默认禁止所有从网上下载的脚本文件（比如 hexo.ps1）直接执行。

解决方法

步骤一：以管理员身份打开 PowerShell

· 点击“开始”，搜索 “PowerShell”。
· 右键点击 “Windows PowerShell”，选择 “以管理员身份运行”。

· 参数解释：
· RemoteSigned：允许运行本地脚本；对于从网上下载的脚本，要求必须有可信发布者的数字签名才可运行。这对我们日常使用（如运行 Hexo）来说是足够安全且方便的。
· Scope CurrentUser：这个更改仅对当前用户生效，不会影响电脑上的其他账户，更安全。
