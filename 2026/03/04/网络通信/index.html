IPv4

一、IPv4地址是什么

IPv4地址是互联网协议的第四版地址，由32位二进制组成，通常写成点分十进制形式，比如 192.168.1.5。

一个IPv4地址分成两部分：

· 网络位：告诉路由器你在哪个“小区”
· 主机位：告诉路由器你是小区里哪栋楼

二、子网掩码（Subnet Mask）

2.1 什么是子网掩码

子网掩码的作用就是区分IP地址的哪部分是网络位，哪部分是主机位。

六、CIDR（无类域间路由）

6.1 为什么要有CIDR

传统的A、B、C类划分太死板：

· 一个C类（/24）只有254个IP，不够用
· 一个B类（/16）有6万多个，又太多浪费

CIDR允许任意长度的网络位，不再被A/B/C类绑死。

6.2 CIDR写法

6.3 CIDR的好处

更灵活地划分IP：

· 一个小公司可能分一个 /27（30个IP）
· 点对点链路用 /30（2个IP）
· 大型数据中心用 /20（4094个IP）

10.2 怎么解决的

IPv6

一、IPv6 是什么

IPv6是互联网协议的第六版，为了解决IPv4地址耗尽问题而设计。

1.1 IPv6 和 IPv4 的核心区别

对比项 IPv4 IPv6
地址长度 32位 128位
地址数量 约42.9亿个 约3.4×10³⁸个（地球上每粒沙子都能分一堆）
表示方式 点分十进制（192.168.1.1） 冒号分十六进制（2001:db8::1）
配置方式 DHCP或手动 SLAAC、DHCPv6或手动
NAT 常用（私网转公网） 基本不用（地址够多）
广播 有广播地址 没有广播，用组播代替
分片 路由器和主机都可分片 只允许主机分片
安全性 可选（IPSec是附加） 原生支持IPSec

三、IPv6地址的三层结构

IPv4有“网络位 + 主机位”，IPv6分得更细：

3.1 全局路由前缀（Global Routing Prefix）

· 通常48位
· 由IANA分配给区域注册机构，再给ISP
· 相当于“国家 + 省份 + 城市”

3.2 子网ID（Subnet ID）

· 通常16位
· 由网络管理员自己划分
· 相当于“小区编号”

3.3 接口ID（Interface ID）

· 64位
· 相当于IPv4的主机位
· 通常由EUI-64（根据MAC地址生成）或随机生成

四、IPv6地址分类（按用途）

4.1 单播地址（Unicast）——一对一通信

类型 前缀 说明 相当于IPv4的
全球单播 2000::/3 公网IP，全球路由 公网IPv4
唯一本地 fc00::/7 私网IP，内网用，但全球唯一 私网IPv4（10.x/172.16.x/192.168.x）
链路本地 fe80::/10 自动生成，只在本网段有效 169.254.x.x
环回地址 ::1/128 你自己 127.0.0.1
未指定地址 ::/128 表示“没有地址”，DHCP用 0.0.0.0

4.2 组播地址（Multicast）——一对多通信

地址 说明
ff02::1 本链路所有节点
ff02::2 本链路所有路由器
ff02::1:3 所有DHCP服务器
ff05::2 本站点所有路由器

IPv6没有广播，都用组播代替。

4.3 任播地址（Anycast）——一对最近的一个

多个接口共享同一个IP，路由器把数据发给“最近”的那个（用于负载均衡、服务发现）。

七、IPv6的网关和默认路由

7.1 默认网关怎么找

IPv6的网关通常就是链路本地地址，而不是全球单播地址。

7.2 为什么网关用链路本地地址

因为：

· 链路本地地址永远不变，不管前缀怎么换
· 路由器换公网前缀时，网关地址不受影响
· 你永远能通过fe80::1找到路由器

八、IPv6的网段和前缀

8.1 最常见的网段大小：/64

IPv6强烈建议每个网段用/64（64位网络位 + 64位主机位），因为：

· SLAAC需要64位接口ID
· 有足够地址支持隐私扩展（随机生成临时地址）
· 邻居发现协议（NDP）优化得好

3.3 微端口驱动（网卡驱动）详解

这是你设备管理器里看到的“网卡驱动”，也是 NDIS 三层中最底层、和硬件直接打交道的那一层。

3.3.1 它做什么？

以你的无线网卡为例，一篇关于无线网卡 NDIS 小端口驱动的论文里提到，驱动初始化时要做的几件事：

  1. 选择媒体类型：无线网卡声明为 NdisMedium802_3（和以太网一样的数据格式）

  4. 选择并注册 I/O 端口、中断
  5. 初始化网卡硬件：设置寄存器、读取永久 MAC 地址
  6. 初始化自定义变量

收发数据时：

· 发送：上层要发数据，NDIS 调用微端口驱动的 Send 函数，驱动把数据传给硬件发出去
· 接收：硬件收到数据产生中断，驱动把数据传给 NDIS，NDIS 再传给上层

3.3.2 它和 NDIS 的关系

NDIS 提供了一整套函数（NdisXxx），微端口驱动调用这些函数来和系统交互。反过来，微端口驱动也要实现一组 MiniportXxx 函数，供 NDIS 在特定事件（比如收到数据包）时调用。

简单说：NDIS 是“规矩”，微端口驱动是“按规矩办事的那个人”。

3.4 你改的那个 Ndisuio 是什么？

Ndisuio 是 NDIS 提供的一个用户模式 I/O 协议驱动。它的作用是：

允许用户模式程序（比如某些网络工具）直接和 NDIS 层交互，绕过上层的 Winsock 和 TDI。

那个逆天软件把它从自动（2）改成手动（3），导致系统启动时 NDIS 层没有完全加载，上层协议和下层硬件之间就“断联”了。这就是为什么你改回来之后，网络系统恢复正常。

四、数据从应用到网卡的完整旅程

以你打开浏览器访问百度为例：

步骤 经过的组件 做什么
1 浏览器 调用 Winsock API
2 Winsock (ws2_32.dll) 格式化请求，传给协议驱动
3 TDI 传输器 (tcpip.sys) 加上 TCP 头、IP 头
4 NDIS 库 (ndis.sys) 把数据传给微端口驱动
5 微端口驱动 控制网卡硬件发出去
6 网卡硬件 转成电信号/无线电发出去

接收数据时，流程相反。

五、几个关键机制

5.1 网络位置（Network Location）

从 Windows 7/2008 R2 开始，系统引入了“网络位置”概念：

· 专用网络：信任的网络（家、公司），允许文件共享
· 公用网络：不信任的网络（咖啡厅、机场），防火墙更严格
· 域网络：加入企业域时自动识别

系统通过 “网络列表服务” 识别每个网络，并分配防火墙规则。

5.2 自动跃点（Automatic Metric）

Windows 会根据网卡速度自动计算“跃点数”（优先级）：

· 速度越快 → 跃点数越低 → 优先级越高
· 以太网（1000Mbps）优先级高于 Wi-Fi（几百Mbps）

这就是为什么同时连有线和无线时，系统默认走以太网。

5.3 APIPA（自动专用IP寻址）

当电脑向 DHCP 服务器要 IP 但没要到时，会自动给自己分配一个 169.254.x.x 的地址。这个地址只能和同一子网的其他设备通信，不能上网。

你之前看到 Wi-Fi 的 IPv4 地址是 169.254.150.229，就是这个原因。

5.4 硬件抽象层（HAL）

HAL 是 Windows 最底层的模块，把所有和硬件相关的代码隔离在这里。这样上层的驱动就不用管你是 Intel 网卡还是 Realtek 网卡，反正通过 HAL 调用就行。

DHCP修复

一、什么是 DHCP

DHCP（Dynamic Host Configuration Protocol，动态主机配置协议），简单说就是网络世界的自动入住前台。

你每次连WiFi或插网线，电脑需要四个东西才能上网：IP地址、子网掩码、默认网关、DNS服务器。DHCP的作用就是自动给你分配这四样，让你不用手动填。

没有DHCP，你就得自己记：

· 这个网络用哪个IP段（192.168.1.x 还是 10.0.0.x）
· 网关地址是多少（大门在哪）
· DNS用哪个（电话本在哪）

二、DHCP 的工作过程（四步握手）

整个过程在几百毫秒内完成，你完全感觉不到。

三、DHCP 出问题会怎样

现象 原因
WiFi图标一直转圈，最后显示“无网络访问” DHCP没拿到IP，无法上网
IP地址变成169.254.x.x 没人回应DHCP请求，系统给自己编了个假IP（只能和自己说话）
能ping通同WiFi的其他设备，但打不开网页 拿到IP了，但没拿到网关或DNS
换了个地方（公司→家），WiFi连上但上不了网 静态IP没改回自动获取，还在用旧网络的配置
两台设备突然同时掉线 IP冲突（有人手动填了正在用的IP）

169.254.x.x 是什么？
这是Windows的自救地址（APIPA）。当电脑喊“给我IP！”没人理，系统就自己编一个。这个IP只能和同网段其他169设备通信，出不了小区（没网关），上不了网。

四、为什么会出现 DHCP 问题

原因 说明

手动改成了静态IP 你或软件（如考试系统）把网卡设成了固定IP
网卡驱动问题 网卡自己坏了，发不出DHCP请求
MAC地址过滤 路由器把你拉黑了，不给你分配IP

你之前遇到的考试系统：它可能直接把你的网卡从“自动获取”改成了“手动填写”，让你用不了网络。

IP获取失败

一、问题现象

· 以太网：插网线能正常上网
· Wi-Fi：能连接上（图标亮），但无法上网，浏览器打不开任何网页

· Wi-Fi 的 IPv4 地址是 169.254.x.x
· 以太网的 IPv4 地址是正常的 192.168.x.x

二、先看懂关键线索：169.254.x.x 是什么意思？

这个地址叫 APIPA 地址（自动专用IP寻址）。

地址类型 来源 能不能上网
192.168.x.x 路由器 DHCP 分配的 能
169.254.x.x 电脑自己临时分配的 不能

出现条件：电脑向路由器要 IP（DHCP 请求），但没要到。

看到这个地址，就能直接定位问题：
Wi-Fi 硬件是好的（所以能给自己分配地址）
DHCP 请求失败（所以没拿到路由器的地址）

3.1 两个网卡共用同一套“上层系统”

· TCP/IP 协议栈是好的
· DHCP 客户端服务是好的
· NDIS 层整体能工作

五、为什么禁用再启用以太网能救 Wi-Fi？

这背后有两个机制：

5.1 协议栈共享

以太网和 Wi-Fi 共用同一套：

· TCP/IP 协议栈
· DHCP 客户端
· NDIS 层

5.2 系统检测到网络变化会重启协议栈

当你禁用以太网时，系统检测到大网络变化，强制重启了整个协议栈和 NDIS 层——顺带把 Wi-Fi 的 DHCP 路径也救活了。

网卡异常

问题：网络图标消失

可能性一、静电

一、为什么静电会导致图标消失？

笔记本内部积累的静电可能导致主板上的网卡硬件处于“假死”状态。长按电源键（关机状态下）可以释放主板残余电荷，让硬件复位。

很多品牌（联想、戴尔、惠普）的官网技术支持里都有这个步骤，叫 “强制放电” 或 “硬件复位”。

二、操作方法

标准步骤

  1. 关机（不是休眠，不是重启，是彻底关机）
  2. 拔掉电源适配器（电源线彻底拔掉）
  3. 拔掉所有外接设备（鼠标、U盘、外接显示器等）
  4. 长按电源键 20-30 秒（不要松手）
    · 过程中电脑不会有任何反应（本来就是关机的）
    · 这是正常的，它在释放主板残余电荷
  5. 松开电源键，插上电源
  6. 正常开机

三、为什么这个方法有用？

笔记本在长时间使用、插拔外设、环境干燥等情况下，主板上可能积累静电。这些静电不会损坏硬件，但可能导致某些芯片（比如网卡）进入“保护状态”或“假死”。

长按电源键（关机状态下）会强制主板放电，把所有残余电荷清空，硬件就能恢复正常状态。

键值名 正常值 被改后的后果
DependOnService 只有 RPCSS 等核心依赖 如果多了 WinHttpAutoProxySvc，可能导致启动失败

操作方法

  1. 进到 Wcmsvc 路径
  2. 找到 DependOnService，双击
  3. 检查列表里有没有 WinHttpAutoProxySvc
  4. 如果有，选中它按 Delete 删掉（不要删别的）
  5. 确定，重启电脑

五、一点提醒

改注册表之前，最好先备份一下：

· 右键要改的项 → 导出
· 存个 .reg 文件，万一改错了能恢复
