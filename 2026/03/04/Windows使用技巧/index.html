二、环境变量是什么

  1. 定义

环境变量是操作系统里的一组键值对，告诉系统去哪里找可执行文件、临时文件夹位置等信息。

  1. 最重要的那个：PATH

PATH 是一个列表，里面放着一堆文件夹路径。当你在命令行输入一个命令（比如 g++），系统会按顺序去这些文件夹里找有没有叫 g++.exe 的文件。

方法A：永久添加（推荐）

    · 此电脑 → 右键属性 → 高级系统设置
    · 点“环境变量”按钮
  3. 编辑 PATH
    · 在“系统变量”里找到 Path，双击编辑
    · 点“新建”，粘贴你的路径

四、进阶：多个版本的同一工具

Windows 里的斜杠——正斜杠/反斜杠

一、问题来源

  1. 现象

你在不同的地方看到这些写法：

到底哪个是对的？为什么同一个路径要写成不同样子？

  1. 历史原因

· Windows：从 DOS 时代继承，用反斜杠 \ 分隔文件夹
· Unix/Linux：用正斜杠 / 分隔
· Web：URL 里用正斜杠
https://example.com/

你现在 straddle 在三个世界之间，所以乱。

二、各场景应该用什么

场景1：文件资源管理器地址栏

用反斜杠 \

这是 Windows 亲儿子，只能用 \。

场景2：命令行（cmd/PowerShell）

两种都行，但建议用反斜杠

cmd 和 PowerShell 都兼容正斜杠，但传统习惯用反斜杠。

场景3：C/C++ 代码里的字符串

必须用双反斜杠 \ 或正斜杠 /

为什么：C++ 里反斜杠 \ 是转义符的开始，比如 \n 是换行。所以要表示真正的反斜杠，得写两个 \。

场景4：CMakeLists.txt 和 JSON 文件

用正斜杠 /

这些文件里反斜杠容易出问题，用正斜杠最安全。

场景5：相对路径

用正斜杠 /

相对路径里的 … 是语法，分隔符用 / 更清晰，各平台都认。

场景6：Windows API

必须用双反斜杠 \

Windows 底层 API 要求反斜杠，但在 C++ 里要写成双的。
