Windows 里的斜杠

一、问题来源

  1. 现象

你在不同的地方看到这些写法：

到底哪个是对的？为什么同一个路径要写成不同样子？

  1. 历史原因

· Windows：从 DOS 时代继承，用反斜杠 \ 分隔文件夹
· Unix/Linux：用正斜杠 / 分隔
· Web：URL 里用正斜杠
https://example.com/

你现在 straddle 在三个世界之间，所以乱。

二、各场景应该用什么

场景1：文件资源管理器地址栏

用反斜杠 \

这是 Windows 亲儿子，只能用 \。

场景2：命令行（cmd/PowerShell）

两种都行，但建议用反斜杠

cmd 和 PowerShell 都兼容正斜杠，但传统习惯用反斜杠。

场景3：C/C++ 代码里的字符串

必须用双反斜杠 \ 或正斜杠 /

为什么：C++ 里反斜杠 \ 是转义符的开始，比如 \n 是换行。所以要表示真正的反斜杠，得写两个 \。

场景4：CMakeLists.txt 和 JSON 文件

用正斜杠 /

这些文件里反斜杠容易出问题，用正斜杠最安全。

场景5：相对路径

用正斜杠 /

相对路径里的 … 是语法，分隔符用 / 更清晰，各平台都认。

场景6：Windows API

必须用双反斜杠 \

Windows 底层 API 要求反斜杠，但在 C++ 里要写成双的。

注册表

一、注册表是什么

  1. 一句话定义

Windows 的配置数据库——所有设置、选项、配置都藏在这里。

  1. 类比

· 如果把电脑比作一个人，注册表就是他的大脑+记忆+习惯
· 系统启动时读它，装软件时写它，运行时随时查它
· 改错了，人就傻了

  1. 物理位置（知道就行，别去翻）

二、为什么需要知道注册表

  1. 软件卸载不干净

· 现象：明明卸载了，重装时报“已安装”
· 原因：软件没了，但注册表里还留着它的信息
· 解决：去注册表里删对应键

  1. 开机启动项太多

· 现象：电脑开机越来越慢
· 原因：很多软件把自己写进开机启动
· 解决：去注册表里删掉不想自启的

  1. 右键菜单太乱

· 现象：装了一堆软件，右键菜单比命还长
· 原因：每个软件都往注册表里加自己的右键项
· 解决：去注册表里删掉不用的

  1. 文件关联错了

· 现象：双击 .txt 文件，它用记事本打开，但你想用 VSCode
· 原因：注册表里记录了默认打开方式
· 解决：改注册表或右键“打开方式”

  1. 某些设置界面找不到

· 现象：想改的东西在设置面板里没有
· 原因：微软藏起来了，但注册表里有开关
· 解决：去注册表里改（需要知道具体键值）

三、注册表长什么样

  1. 打开方式

（Win + R 输入 regedit 回车）

  1. 结构（树形，和文件管理器一样）
  1. 五个根键的作用

根键 缩写 存什么
HKEY_CLASSES_ROOT HKCR 文件类型、右键菜单、COM组件
HKEY_CURRENT_USER HKCU 当前用户的设置（主题、壁纸、软件配置）
HKEY_LOCAL_MACHINE HKLM 本机所有用户的设置（硬件、系统、全局软件）
HKEY_USERS HKU 所有用户的配置（一般不用管）
HKEY_CURRENT_CONFIG HKCC 当前硬件配置（一般不用管）

  1. 键和值

· 键（Key）：左边那些文件夹，像 HKEY_CURRENT_USER\Software\Microsoft
· 值（Value）：右边那些，存具体数据
· 值类型：字符串、数字、二进制

  1. 右键菜单（文件）

Windows网络机制

一、Windows 网络整体架构（五层模型）

Windows 的网络实现大致分为五层，从应用程序到硬件：

二、核心概念逐个拆解

  1. 网络适配器（网卡）

就是你在设备管理器里看到的那些：

· 有线网卡：Realtek PCIe GbE Family Controller
· 无线网卡：MediaTek Wi-Fi 6 MT7921

每个网卡是一个硬件设备，有自己的驱动程序。

  1. 协议栈（TCP/IP）

协议栈是网络的“规则集”。你的浏览器发一个请求，要经过层层封装：

Windows 从 Windows 7/2008 R2 开始，原生支持 IPv4 和 IPv6 双协议栈并行运行。

  1. Winsock（Windows套接字）

这是应用程序和网络协议之间的“翻译官”：

Winsock 的好处是：应用程序不需要知道底层是 Wi-Fi 还是有线，只管调用接口。

  1. TDI（传输驱动程序接口）

TDI 是协议驱动向上层提供服务的标准接口。它规定了：

· 网络请求怎么格式化成 IRP（I/O请求包）
· 网络地址怎么分配
· 通信连接怎么管理

你电脑里的 tcpip.sys 就是一个 TDI 传输器。

三、NDIS 层（核心中的核心）

3.1 NDIS 是什么？

NDIS（网络驱动程序接口规范）是微软和 3COM 在 1989 年联合开发的规范，目的是让协议驱动和网卡驱动能互相通信，而不需要知道对方的具体细节。

3.3 微端口驱动（网卡驱动）详解

这是你设备管理器里看到的“网卡驱动”，也是 NDIS 三层中最底层、和硬件直接打交道的那一层。

3.3.1 它做什么？

以你的无线网卡为例，一篇关于无线网卡 NDIS 小端口驱动的论文里提到，驱动初始化时要做的几件事：

  1. 选择媒体类型：无线网卡声明为 NdisMedium802_3（和以太网一样的数据格式）
  2. 从注册表读取网卡地址
  3. 注册适配器类型（比如 PCMCIA、PCIe）
  4. 选择并注册 I/O 端口、中断
  5. 初始化网卡硬件：设置寄存器、读取永久 MAC 地址
  6. 初始化自定义变量

收发数据时：

· 发送：上层要发数据，NDIS 调用微端口驱动的 Send 函数，驱动把数据传给硬件发出去
· 接收：硬件收到数据产生中断，驱动把数据传给 NDIS，NDIS 再传给上层

3.3.2 它和 NDIS 的关系

NDIS 提供了一整套函数（NdisXxx），微端口驱动调用这些函数来和系统交互。反过来，微端口驱动也要实现一组 MiniportXxx 函数，供 NDIS 在特定事件（比如收到数据包）时调用。

简单说：NDIS 是“规矩”，微端口驱动是“按规矩办事的那个人”。

3.4 你改的那个 Ndisuio 是什么？

Ndisuio 是 NDIS 提供的一个用户模式 I/O 协议驱动。它的作用是：

允许用户模式程序（比如某些网络工具）直接和 NDIS 层交互，绕过上层的 Winsock 和 TDI。

那个逆天软件把它从自动（2）改成手动（3），导致系统启动时 NDIS 层没有完全加载，上层协议和下层硬件之间就“断联”了。这就是为什么你改回来之后，网络系统恢复正常。

四、数据从应用到网卡的完整旅程

以你打开浏览器访问百度为例：

步骤 经过的组件 做什么
1 浏览器 调用 Winsock API
2 Winsock (ws2_32.dll) 格式化请求，传给协议驱动
3 TDI 传输器 (tcpip.sys) 加上 TCP 头、IP 头
4 NDIS 库 (ndis.sys) 把数据传给微端口驱动
5 微端口驱动 控制网卡硬件发出去
6 网卡硬件 转成电信号/无线电发出去

接收数据时，流程相反。

五、几个关键机制

5.1 网络位置（Network Location）

从 Windows 7/2008 R2 开始，系统引入了“网络位置”概念：

· 专用网络：信任的网络（家、公司），允许文件共享
· 公用网络：不信任的网络（咖啡厅、机场），防火墙更严格
· 域网络：加入企业域时自动识别

系统通过 “网络列表服务” 识别每个网络，并分配防火墙规则。

5.2 自动跃点（Automatic Metric）

Windows 会根据网卡速度自动计算“跃点数”（优先级）：

· 速度越快 → 跃点数越低 → 优先级越高
· 以太网（1000Mbps）优先级高于 Wi-Fi（几百Mbps）

这就是为什么同时连有线和无线时，系统默认走以太网。

5.3 APIPA（自动专用IP寻址）

当电脑向 DHCP 服务器要 IP 但没要到时，会自动给自己分配一个 169.254.x.x 的地址。这个地址只能和同一子网的其他设备通信，不能上网。

你之前看到 Wi-Fi 的 IPv4 地址是 169.254.150.229，就是这个原因。

5.4 硬件抽象层（HAL）

HAL 是 Windows 最底层的模块，把所有和硬件相关的代码隔离在这里。这样上层的驱动就不用管你是 Intel 网卡还是 Realtek 网卡，反正通过 HAL 调用就行。

故障排查

问题一：

一、问题现象

· 以太网：插网线能正常上网
· Wi-Fi：能连接上（图标亮），但无法上网，浏览器打不开任何网页

运行 ipconfig 后看到：

· Wi-Fi 的 IPv4 地址是 169.254.x.x
· 以太网的 IPv4 地址是正常的 192.168.x.x

二、先看懂关键线索：169.254.x.x 是什么意思？

这个地址叫 APIPA 地址（自动专用IP寻址）。

地址类型 来源 能不能上网
192.168.x.x 路由器 DHCP 分配的 能
169.254.x.x 电脑自己临时分配的 不能

出现条件：电脑向路由器要 IP（DHCP 请求），但没要到。

看到这个地址，就能直接定位问题：
Wi-Fi 硬件是好的（所以能给自己分配地址）
DHCP 请求失败（所以没拿到路由器的地址）

三、为什么以太网能拿到 IP，Wi-Fi 拿不到？

3.1 两个网卡共用同一套“上层系统”

以太网和 Wi-Fi 虽然硬件独立，但共享以下组件：

所以当以太网能正常上网时，说明：

· TCP/IP 协议栈是好的
· DHCP 客户端服务是好的
· NDIS 层整体能工作

四、解决方法：禁用再启用以太网

  1. Win + R → ncpa.cpl
  2. 右键以太网 → 禁用
  3. 等 10 秒 → 右键以太网 → 启用

结果：Wi-Fi 突然能拿到正常 IP 了！

五、为什么禁用再启用以太网能救 Wi-Fi？

这背后有两个机制：

5.1 协议栈共享

以太网和 Wi-Fi 共用同一套：

· TCP/IP 协议栈
· DHCP 客户端
· NDIS 层

5.2 系统检测到网络变化会重启协议栈

当你禁用以太网时，系统检测到大网络变化，强制重启了整个协议栈和 NDIS 层——顺带把 Wi-Fi 的 DHCP 路径也救活了。

问题二、网络图标消失

可能性一、静电

一、为什么静电会导致图标消失？

笔记本内部积累的静电可能导致主板上的网卡硬件处于“假死”状态。长按电源键（关机状态下）可以释放主板残余电荷，让硬件复位。

很多品牌（联想、戴尔、惠普）的官网技术支持里都有这个步骤，叫 “强制放电” 或 “硬件复位”。

二、操作方法

标准步骤

  1. 关机（不是休眠，不是重启，是彻底关机）
  2. 拔掉电源适配器（电源线彻底拔掉）
  3. 拔掉所有外接设备（鼠标、U盘、外接显示器等）
  4. 长按电源键 20-30 秒（不要松手）
    · 过程中电脑不会有任何反应（本来就是关机的）
    · 这是正常的，它在释放主板残余电荷
  5. 松开电源键，插上电源
  6. 正常开机

三、为什么这个方法有用？

笔记本在长时间使用、插拔外设、环境干燥等情况下，主板上可能积累静电。这些静电不会损坏硬件，但可能导致某些芯片（比如网卡）进入“保护状态”或“假死”。

长按电源键（关机状态下）会强制主板放电，把所有残余电荷清空，硬件就能恢复正常状态。

可能性二、服务没启动

一、核心服务：WLAN AutoConfig

Windows 的无线网络功能由一个关键服务控制：WLAN AutoConfig（无线网络自动配置服务）。

如果这个服务没启动，后果包括：

· Wi-Fi 图标消失
· 无法扫描 Wi-Fi 网络
· 无法连接 Wi-Fi

二、怎么检查和启动

步骤 1：打开服务管理器

Win + R → 输入 services.msc → 回车

步骤 2：找到 WLAN AutoConfig

在列表里按 W 键快速定位，找到 WLAN AutoConfig。

步骤 3：检查状态

双击打开，看两个地方：

检查项 正确状态 如果不对怎么办
启动类型 自动 改成“自动”，点应用
服务状态 正在运行 点“启动”

步骤 4：如果启动失败

点“启动”时报错，可能是：

· 依赖服务没启动（Ndisuio、RpcSs 等）
· 注册表被改过

四、如果 WLAN AutoConfig 正常，但图标还没回来

再检查一个相关的服务：Network Location Awareness

这个服务负责识别网络类型（专用/公用），有时也会影响图标显示。

同样在服务列表里找到 Network Location Awareness（或 NlaSvc）：

· 启动类型：自动
· 服务状态：正在运行

可能性三、驱动异常

一、什么时候该试这个方法？

· 已经试过静电释放、服务检查，都没用
· 图标消失，Wi-Fi 完全搜不到
· 设备管理器里能看到问题

二、驱动异常的几种表现

打开设备管理器（Win + X → 设备管理器），展开 “网络适配器”，找你的无线网卡（MediaTek/Realtek/Intel）：

表现 含义 怎么办
有向下箭头 网卡被禁用了 右键 → 启用设备
黄色感叹号 驱动故障 右键 → 更新驱动 / 卸载重装
根本找不到 硬件没被识别 扫描硬件改动 / 重装主板驱动
名字是“以太网控制器” 驱动没装上 去官网下驱动手动装

三、具体操作

3.1 如果网卡被禁用

右键 → “启用设备”
等几秒，右下角图标可能就回来了。

3.2 如果驱动有黄色感叹号

方法一：更新驱动

  1. 右键 → “更新驱动程序”
  2. 选 “自动搜索驱动程序”
  3. 等它搜完，如果找到就装上

方法二：卸载重装（更彻底）

  1. 右键 → “卸载设备”
  2. 勾选“删除此设备的驱动程序软件”
  3. 确定，重启电脑
  4. 重启后 Windows 会自动重装驱动

3.3 如果网卡完全找不到

  1. 在设备管理器顶部点 “操作” → “扫描检测硬件改动”
  2. 等它扫完，看看网卡会不会出现
  3. 如果还是找不到，可能是：
    · BIOS 里被禁用了
    · 硬件故障
    · 主板驱动没装全

四、BIOS 里看一眼（进阶）

有些软件会改 BIOS 设置，导致网卡被硬件级禁用：

  1. 重启电脑，开机时按 F2 或 Del 进 BIOS
  2. 找 “Onboard LAN”、“Wireless LAN” 或 “WLAN Device”
  3. 确保是 Enabled（开启状态）
  4. 保存退出重启

可能性四、注册表被改

一、什么时候该试这个方法？

· 已经试过静电释放、服务检查、驱动重装，都没用
· 图标消失，Wi-Fi 功能异常
· 你怀疑之前装的某个软件改了系统底层

二、为什么注册表会影响网络图标？

网络图标不是单纯的一个“开关”，它依赖底层服务的正常运行。如果某个关键服务的注册表配置被改，服务启动失败，图标就会消失。

最常见被改的地方有两个：

注册表路径 作用 被改后的后果
Services\Ndisuio NDIS 用户态协议驱动 网络协议栈加载不全
Services\Wcmsvc 连接管理器 网络连接状态无法识别

你之前遇到的就是第一种。

三、检查位置一：Ndisuio

路径

检查项

键值名 正常值 被改后的值 含义
Start 2 3 或 4 2=自动，3=手动，4=禁用

操作方法

  1. Win + R → regedit → 回车
  2. 粘贴路径到地址栏，回车
  3. 找到 Start，双击
  4. 如果不是 2，改成 2，点确定
  5. 重启电脑

四、检查位置二：Wcmsvc

这个服务负责管理网络连接状态，如果它的依赖项被改，也可能导致图标消失。

路径

检查项

键值名 正常值 被改后的后果
DependOnService 只有 RPCSS 等核心依赖 如果多了 WinHttpAutoProxySvc，可能导致启动失败

操作方法

  1. 进到 Wcmsvc 路径
  2. 找到 DependOnService，双击
  3. 检查列表里有没有 WinHttpAutoProxySvc
  4. 如果有，选中它按 Delete 删掉（不要删别的）
  5. 确定，重启电脑

五、一点提醒

改注册表之前，最好先备份一下：

· 右键要改的项 → 导出
· 存个 .reg 文件，万一改错了能恢复

DNS修复

一、什么是 DNS

DNS（Domain Name System，域名系统），简单说就是互联网的电话本。

你平时访问网站记的是域名，比如 github.com、google.com，但电脑真正要找的是IP 地址，比如 140.82.112.4。DNS 的作用就是把 github.com 翻译成对应的 IP 地址，让你的电脑知道该往哪连。

二、DNS 的工作过程（以访问 GitHub 为例）

整个过程在几十毫秒内完成，你完全感觉不到。

三、DNS 出问题会怎样

现象 原因
浏览器打不开网站，显示乱码或超时 DNS 返回了错误的 IP，或者根本没返回
微信/QQ 能上，但网页打不开 聊天软件用 IP 直连，不受 DNS 影响
hexo deploy 报错 502 或 Connection reset Git 要连 GitHub，但 DNS 给了错误的路
能 commit 但不能 push 本地提交不依赖网络，推送到远程才需要 DNS

四、为什么会出现 DNS 问题

原因 说明
DNS 缓存错误 电脑里存了旧的 IP 地址，实际 GitHub 已经换地址了
DNS 服务器污染 你用的 DNS 服务器（比如运营商默认的）故意返回错误 IP
本地 hosts 文件被改 有人手动把 github.com 指向了错误地址
网络限制 校园网、公司网故意让 DNS 解析失败

问题现象

在 PowerShell 里运行 hexo 命令时，出现以下报错：

无法加载文件 …hexo.ps1，因为在此系统上禁止运行脚本。

问题原因

这是 Windows 的 执行策略（Execution Policy） 在起作用。它就像系统的一个安全开关，为了防止恶意脚本运行，默认禁止所有从网上下载的脚本文件（比如 hexo.ps1）直接执行。

解决方法

步骤一：以管理员身份打开 PowerShell

· 点击“开始”，搜索 “PowerShell”。
· 右键点击 “Windows PowerShell”，选择 “以管理员身份运行”。
