生成（Generate）

这一步不是编译代码，而是准备编译环境。

任务

把 CMakeLists.txt 翻译成构建工具能读懂的配置文件。

输入

· CMakeLists.txt
· 你指定的生成器类型

输出

· Makefile（给 make 用）
· .sln + .vcxproj（给 MSBuild 用）
· build.ninja（给 ninja 用）

你用的命令

构建（Build）

这一步是编译和链接的调度中心。

任务

读生成器输出的文件，根据依赖关系，调用编译器和链接器，只编译有变动的部分。

输入

· 生成器输出的文件（Makefile/.sln/build.ninja）
· 你的源码

输出

· 调用编译器的命令
· 调用链接器的命令
· 最终生成 .exe

命令

编译（Compile）

这是真正处理你的代码的一步。

任务

把 .cpp 文件翻译成机器码（但还没拼成完整程序），生成目标文件（.obj/.o）。

输入

· .cpp 文件
· 头文件（.h/.hpp）
· 编译选项（优化级别、宏定义等）

输出

· .obj（Windows）
· .o（Linux/macOS）

编译器内部做了哪些事

  1. 预处理：处理 #include、#define、宏展开
  2. 词法分析：把 int a = b + c 拆成单词（token）
  3. 语法分析：检查语法，生成语法树
  4. 语义分析：检查类型（比如不能把 string 赋值给 int）
  5. 中间代码生成：转成和机器无关的 IR（Intermediate Representation）
  6. 优化：删掉无用代码、循环展开、内联等
  7. 汇编：把 IR 转成汇编语言
  8. 汇编器：把汇编转成机器码，输出 .obj/.o

手动编译命令

链接（Link）

任务

把多个目标文件（.obj/.o）和库文件（.lib/.a）拼成一个完整的可执行文件。

输入

· 你的目标文件（main.obj、utils.obj）
· 静态库（.lib/.a）
· 动态库（.dll/.so）的导入库

输出

· .exe（Windows）
· 可执行文件（Linux/macOS 无后缀）

链接做了哪些事

  1. 符号解析：找到所有函数和变量的定义
  2. 地址重定位：给每个符号分配内存地址
  3. 合并段：把所有 .obj 的代码段合并，数据段合并
  4. 库的静态链接：把用到的库函数代码复制进 exe
  5. 动态库标记：记录需要哪些 .dll/.so
